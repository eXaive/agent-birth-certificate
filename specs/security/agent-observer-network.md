5. Observer Data Stream (ODS)

All Observers publish to a protected internal channel:

ODS://safezone/<cluster_id>/<observer_id>


This real-time data includes:

timing irregularities

ODS is write-only, ensuring Observers cannot be corrupted by feedback.
